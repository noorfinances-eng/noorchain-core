NOORCHAIN — Chemin de Migration du Genesis

Version 1.1
Dernière mise à jour : 2025-XX-XX

1. Objet du document

Le Chemin de Migration du Genesis définit comment NOORCHAIN évolue :

du bloc genesis,

vers un mainnet stable,

puis vers un réseau PoSS activé,

puis vers un écosystème gouverné et évolutif.

Il garantit une progression sûre, prévisible et conforme au cadre Legal Light CH.

2. Philosophie de migration

NOORCHAIN applique une philosophie de migration progressive :

Le genesis avant tout

La stabilité avant toute activation

Activation PoSS uniquement confirmée

Pas d’expansion immédiate des validateurs

Pas de mécanismes de trésorerie

Gouvernance activable mais non obligatoire

3. Phase 0 — Genesis (T0)

Au lancement, le réseau démarre avec :

PoSS désactivé

EVM vide

Aucun contrat préchargé

Set de validateurs minimal

Allocations initiales fixes

Gouvernance limitée mais active

Aucun mint n’a lieu au genesis.

4. Phase 1 — Période de stabilité (T0 → T0 + 3 mois)

Restrictions :

PoSS reste désactivé

Pas d’upgrade de module

Pas de nouveaux validateurs

Pas de distributions supplémentaires

Surveillance renforcée

Outils activés :

explorateurs

indexeurs

tableau de gouvernance

audit de supply

Objectif : valider la stabilité du réseau.

5. Phase 2 — Activation de PoSS (via gouvernance)

PoSS est activé seulement après :

rapport de stabilité complet

validation du Multi-sig (3/5)

validation de la Fondation

annonce publique

mise à jour des documents

Effets :

PoSSEnabled = true

Mint depuis la réserve PoSS

Limites journalières en vigueur

6. Phase 3 — Expansion du système des Curateurs

Après activation de PoSS :

ajout de Curateurs (Bronze / Argent / Or)

augmentation progressive de la capacité

distribution élargie

votes consultatifs possibles

7. Phase 4 — Croissance de l’écosystème EVM

Lorsque PoSS devient stable :

déploiement de smart contracts

lancement de dApps (Hub, Pay, Studio)

onboarding partenaires

Chaque déploiement doit être approuvé par la gouvernance.

8. Phase 5 — Expansion du set de validateurs

Extensions possibles quand :

répartition du stake sécurisée

approbation governance

performance validée

9. Phase 6 — Activation governance secondaire

Gouvernance débloque progressivement :

changements de paramètres

upgrades de modules

rôles consultatifs

participation des Curateurs

Certaines règles restent immuables.

10. Phase 7 — Mainnet pleinement opérationnel

Le réseau est considéré mature lorsque :

PoSS est stable depuis 6 mois

le cycle governance fonctionne

les dApps sont actives

l’économie est prouvée

aucune vulnérabilité

11. Invariants de sécurité (jamais modifiables)

Resteront toujours immuables :

supply fixe

split PoSS 70/30

halving 8 ans

modèle 5/5/5/5/80

restrictions Legal Light

aucune garde de fonds
